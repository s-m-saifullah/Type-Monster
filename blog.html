Both localStorage and sessionStorage are browser based data storing mechanism. Even though they work allmost similarly, there is one key difference between them.
localStorage data has no expiration date. They are available even if the browser is closed. On the other hand, sessionStoage has expiration time. They will be lost as soon as the tab or browser is closed.

Scope is a feature of variables. There are two types of scope: global scope and block scope or local scope.
If a variable is declared in the global scope that is called a global variable. A global variable can be defined anywhere in the JavaScript code and can be accessed from anywhere. global variable can be declared with var keyword
If a variable is declared in the block/ local scope that is called a local variable. A local variable can only be visible within two braces: function, if block, else block etc. Local variables can be declared with let or const keyword.

JavaScript is a single threaded programming language. That means it cannot run more than one command simultaneously. So, if some code takes time to execute, codes below that would stuck. To solve this issue event loop was introduced. When a certain code takes time to execute that code/ function moves to thread pool. After the code excutes in the thread pool it is moved to the callback queue. When the call stack executes all the codes it takes input from the call back queue. Event loop has the responsibility to push items from call back queue to call stack.

undefined is data type in JavaScript. It can be produced in different ways:
  • If a variable is being used without initialization.
  • If a function has no return.
  • If a function parameter isn't passed when the function is called.
  • If we try to access a property of an object or an array which is not there.
  • If undefined is set as a value of a variable.
  • If we try to delete an element of an array.
